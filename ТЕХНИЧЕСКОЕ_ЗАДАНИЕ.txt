Техническое задание

1.1. Название проекта. Назначение веб-сайта
Разработка веб-сайта для ГБУЗ ПО «Островская межрайонная больница».
Назначением веб-сайта является: Предоставление пользователям удобной платформы для получения информации о больнице, просмотра новостей, записи на приём к врачам, оставления отзывов и обратной связи.

1.2. Требования к веб-сайту
	Разработать информационный веб-сайт медицинского учреждения с возможностью: просмотра информации о больнице, просмотра новостей и статей блога, записи на приём к врачам, оставления комментариев к статьям, обратной связи с администрацией.

1.2.1 Функциональные требования 
	Проект обеспечивает три уровня доступа к своим ресурсам в соответствии с возможными ролями посетителей: 
•	Гость имеет возможность:
o	Просматривать следующие страницы сайта:
	Главная;
	О больнице;
	Контакты и приём;
	Записи на приём;
	Обратная связь;
	Новости (блог) без возможности оставлять комментарии к статьям блога;
	Полезные ссылки;
	Видео;
o	Зарегистрироваться и авторизоваться в системе;
o	Записаться на приём к врачу через форму на странице «Контакты и приём»;
o	Просматривать список всех записей на приём на странице «Записи на приём»;
o	Отправить отзыв через форму обратной связи.

•	Клиент (авторизованный пользователь) имеет возможность:
o	Выйти из системы;
o	Просматривать следующие страницы сайта:
	Главная;
	О больнице;

o	Добавлять комментарии к статьям на странице «Новости»;
o	Записаться на приём к врачу через форму на странице «Контакты и приём»;
o	Просматривать список всех записей на приём на странице «Записи на приём»;
o	Отправить отзыв через форму обратной связи.

•	Администратор (суперпользователь) имеет возможность:
o	Выйти из системы;
o	Просматривать следующие страницы сайта:
	Главная;
	О больнице;

1.2.3. Технические требования
•	Веб-сайт должен быть разработан с использованием фреймворка Django;
•	База данных должна быть реализована с использованием Django ORM;
•	Интерфейс должен быть адаптивным и корректно отображаться на различных устройствах;
•	Веб-сайт должен использовать Bootstrap для стилизации;
•	Все формы должны иметь валидацию на стороне сервера;
•	Система должна обеспечивать безопасность данных пользователей;
•	Пароли должны храниться в зашифрованном виде;
•	Система должна предотвращать двойную запись на одно и то же время к одному врачу;
•	Система должна валидировать даты записи (нельзя записаться на прошедшую дату);
•	Врачи, помеченные как неактивные, не должны отображаться в форме записи на приём.

1.2.4. Требования к базе данных
База данных должна содержать следующие таблицы:
•	User (встроенная модель Django) – пользователи системы;
•	Posts (Blog) – статьи блога с полями: id, title, description, content, posted, image, video_url, author_id;
•	Comments – комментарии к статьям с полями: id, text, date, author_id, post_id;
•	Feedback – отзывы пользователей с полями: id, name, city, job, gender, design, content, notice, contact, email, message, date;
•	Doctors – врачи с полями: id, full_name, specialty, office, work_schedule, is_active;
•	Appointments – записи на приём с полями: id, patient_name, doctor_id, appointment_date, appointment_time, phone, email, comment, created_at, is_confirmed;
•	Связи между таблицами:
	Posts.author_id → User.id (ForeignKey, SET_NULL);
	Comments.author_id → User.id (ForeignKey, CASCADE);
	Comments.post_id → Posts.id (ForeignKey, CASCADE);
	Appointments.doctor_id → Doctors.id (ForeignKey, CASCADE);
•	Ограничения уникальности:
	Уникальная комбинация (doctor_id, appointment_date, appointment_time) в таблице Appointments;
	Уникальный заголовок статьи для каждой даты в таблице Posts.
