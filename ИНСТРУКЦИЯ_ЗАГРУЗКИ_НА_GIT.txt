ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ ПРОЕКТА НА GIT
ГБУЗ ПО «Островская межрайонная больница»
==========================================

ВАРИАНТ 1: ЧЕРЕЗ КОМАНДНУЮ СТРОКУ (РЕКОМЕНДУЕТСЯ)
--------------------------------------------------

ШАГ 1: Установка Git (если не установлен)
------------------------------------------
1. Скачайте Git с официального сайта: https://git-scm.com/download/win
2. Установите Git, следуя инструкциям установщика
3. Проверьте установку, выполнив в командной строке:
   git --version

ШАГ 2: Настройка Git (первый раз)
----------------------------------
Откройте командную строку (PowerShell или CMD) и выполните:

git config --global user.name ZenyataFE
git config --global user.email nik.egorenkov2004@mail.ru

ШАГ 3: Инициализация репозитория
---------------------------------
1. Откройте командную строку
2. Перейдите в папку проекта:
   cd "d:\Django VS\DjangoWebProject_Theme6-main"

3. Инициализируйте Git репозиторий:
   git init

ШАГ 4: Проверка .gitignore
---------------------------
Убедитесь, что файл .gitignore содержит все необходимые исключения.
Файл уже настроен и включает:
- Виртуальное окружение (env/)
- База данных (db.sqlite3)
- Медиа файлы (/media/)
- Файлы Visual Studio (.vs/)
- Кэш Python (__pycache__/)

ШАГ 5: Добавление файлов
-------------------------
Добавьте все файлы проекта:

git add .

Проверьте, какие файлы будут добавлены:

git status

ШАГ 6: Создание первого коммита
--------------------------------
git commit -m "Initial commit: Django проект для ГБУЗ ПО Островская МБ"

ШАГ 7: Создание репозитория на GitHub
--------------------------------------
1. Зайдите на https://github.com
2. Войдите в свой аккаунт (или создайте новый)
3. Нажмите кнопку "+" в правом верхнем углу
4. Выберите "New repository"
5. Заполните:
   - Repository name: DjangoWebProject_Theme6 (или другое имя)
   - Description: Веб-сайт для ГБУЗ ПО «Островская межрайонная больница»
   - Выберите Public или Private
   - НЕ ставьте галочки на "Initialize with README" (репозиторий уже инициализирован)
6. Нажмите "Create repository"

ШАГ 8: Подключение к удаленному репозиторию
--------------------------------------------
После создания репозитория GitHub покажет инструкции. Выполните:

git remote add origin https://github.com/ВАШ_USERNAME/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

Например:
git remote add origin https://github.com/username/DjangoWebProject_Theme6.git

ШАГ 9: Загрузка проекта на GitHub
-----------------------------------
git branch -M main
git push -u origin main

Если потребуется авторизация:
- GitHub может запросить логин и пароль
- Или использовать Personal Access Token (рекомендуется)

Создание Personal Access Token:
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token (classic)
3. Выберите права: repo (полный доступ к репозиториям)
4. Скопируйте токен
5. При push используйте токен вместо пароля

ВАРИАНТ 2: ЧЕРЕЗ GITHUB DESKTOP (ПРОЩЕ)
----------------------------------------

ШАГ 1: Установка GitHub Desktop
--------------------------------
1. Скачайте GitHub Desktop: https://desktop.github.com/
2. Установите и войдите в свой аккаунт GitHub

ШАГ 2: Создание репозитория
----------------------------
1. В GitHub Desktop: File → Add Local Repository
2. Выберите папку проекта: d:\Django VS\DjangoWebProject_Theme6-main
3. Если Git не инициализирован, нажмите "Create a Repository"
4. Заполните:
   - Name: DjangoWebProject_Theme6
   - Description: Веб-сайт для ГБУЗ ПО «Островская межрайонная больница»
   - Local path: d:\Django VS\DjangoWebProject_Theme6-main

ШАГ 3: Первый коммит
---------------------
1. В GitHub Desktop вы увидите все измененные файлы
2. Внизу введите сообщение коммита: "Initial commit: Django проект для ГБУЗ ПО Островская МБ"
3. Нажмите "Commit to main"

ШАГ 4: Публикация на GitHub
----------------------------
1. Нажмите кнопку "Publish repository"
2. Выберите:
   - Keep this code private (если хотите приватный репозиторий)
   - Или оставьте пустым для публичного
3. Нажмите "Publish Repository"

ВАРИАНТ 3: ЧЕРЕЗ VISUAL STUDIO CODE
-------------------------------------

ШАГ 1: Установка расширения Git
-------------------------------
1. Откройте проект в VS Code
2. Установите расширение "Git" (обычно уже установлено)

ШАГ 2: Инициализация
---------------------
1. Откройте терминал в VS Code (Ctrl + `)
2. Выполните:
   git init
   git add .
   git commit -m "Initial commit"

ШАГ 3: Публикация
------------------
1. Нажмите на иконку Source Control (левая панель)
2. Нажмите "..." → "Publish to GitHub"
3. Следуйте инструкциям

ВАЖНЫЕ ЗАМЕЧАНИЯ
-----------------

1. НЕ ЗАГРУЖАЙТЕ:
   ✓ Виртуальное окружение (env/)
   ✓ Базу данных (db.sqlite3)
   ✓ Секретные ключи (settings_production.py - если содержит секреты)
   ✓ Медиа файлы пользователей (/media/)
   ✓ Файлы Visual Studio (.vs/)

2. ОБЯЗАТЕЛЬНО ЗАГРУЗИТЕ:
   ✓ Весь код приложения (app/)
   ✓ Шаблоны (templates/)
   ✓ Статические файлы (static/)
   ✓ Миграции (migrations/)
   ✓ requirements.txt
   ✓ manage.py
   ✓ settings.py
   ✓ urls.py
   ✓ .gitignore
   ✓ README.md
   ✓ Все документы и схемы

3. ПЕРЕД ЗАГРУЗКОЙ ПРОВЕРЬТЕ:
   - Убедитесь, что в settings.py нет секретных ключей
   - Создайте файл settings_local.py для локальных настроек (если нужно)
   - Проверьте, что .gitignore правильно настроен

4. ПОСЛЕ ЗАГРУЗКИ:
   - Создайте README.md с описанием проекта (если еще нет)
   - Добавьте инструкции по установке
   - Укажите зависимости в requirements.txt

ПОЛЕЗНЫЕ КОМАНДЫ GIT
--------------------

# Проверить статус
git status

# Добавить все файлы
git add .

# Добавить конкретный файл
git add app/views.py

# Создать коммит
git commit -m "Описание изменений"

# Посмотреть историю коммитов
git log

# Отправить изменения на GitHub
git push

# Получить изменения с GitHub
git pull

# Создать новую ветку
git branch название_ветки

# Переключиться на ветку
git checkout название_ветки

# Посмотреть удаленные репозитории
git remote -v

# Обновить информацию о удаленном репозитории
git fetch

РЕКОМЕНДАЦИИ
------------

1. Делайте коммиты часто с понятными сообщениями
2. Используйте .gitignore для исключения ненужных файлов
3. Не коммитьте секретные ключи и пароли
4. Создайте хороший README.md с описанием проекта
5. Используйте ветки для разработки новых функций

СОЗДАНИЕ README.MD (РЕКОМЕНДУЕТСЯ)
-----------------------------------

Создайте файл README.md в корне проекта с описанием:

# ГБУЗ ПО «Островская межрайонная больница»

Веб-сайт для медицинского учреждения на Django.

## Возможности

- Информационные страницы о больнице
- Блог с новостями и комментариями
- Система записи на приём к врачам
- Обратная связь
- Административная панель

## Установка

1. Клонируйте репозиторий
2. Создайте виртуальное окружение
3. Установите зависимости: pip install -r requirements.txt
4. Выполните миграции: python manage.py migrate
5. Создайте суперпользователя: python manage.py createsuperuser
6. Запустите сервер: python manage.py runserver

## Технологии

- Django
- Bootstrap
- SQLite
